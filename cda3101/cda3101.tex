\section*{Chapter 1 - Basic Computer Structure}

\subsection*{Von Neumann Architecture}

\textbf{Von Neumann Architecture} is a computer design where instructions and data are stored in the same memory space.

\begin{itemize}
    \item \textbf{Fixed Program Computers} - Specific function, immutability (cannot be reprogrammed).
    \item \textbf{Stored Program Computers} - Versatile, programmed to carry out many different tasks; applications are stored on them.
\end{itemize}

3 components make up a \textit{Von Neumann computer}:
\begin{itemize}
    \item \textbf{CPU (Central Processing Unit)} - Handles instructions from programs, processes data, stores information, and produces results.
        \begin{itemize}
            \item \textbf{CU (Control Unit)} - Manages how the processor works by sending control signals, controls input and output operations, and fetches the instructions from memory for execution.
            \item \textbf{ALU (Arithmetic and Logic Unit)} - Performs arithmetic operations like \textit{addition} and \textit{subtraction}, logical comparisons such as \textit{comparisons}, and tasks like shifting bits.
            \item \textbf{Registers} - Fastest type of memory located inside the CPU. They temporarily store information that the processor is currently working on. % TODO: Add different types of registers
            \item \textbf{Bus} - Communication system that transfers data, addresses, and control signals between the CPU, memory, and I/O devices. In \textit{Von Neumann architecture}, a single bus is shared for both data and instructions.
        \end{itemize}
\end{itemize}

\textbf{Key Characteristics}

\begin{enumerate}
    \item \textbf{Single Memory} - Data and program instructions stored in the same memory.
    \item \textbf{Shared Bus} - Single bus used for transferring data addresses, and control signals.
    \item \textbf{Sequential Execution} - Instructions are executed one at a time in a sequential manner.
\end{enumerate}

\textbf{Von Neumann Bottleneck}

Because tasks are completed sequentially due to the \textit{shared bus}, this is an innate architectural bottleneck of the design of \textit{Von Neumann architecture}, which hold back the competence of the CPU.
\medskip

\textbf{Applications}

\textit{Von Neumann architecture} is the foundation of most modern computing systems. This includes personal computers, embedded systems, servers, mobile devices, and gaming consoles.

\subsection*{Harvard Architecture}

\textbf{Harvard Architecture} is a computer deisgn model where program instructions and data are stored in separate memory units accessed through independent buses.
\medskip

\textbf{Buses} - used as signal pathways.

\begin{itemize}
    \item \textbf{Data Bus} - carries data between main components.
    \item \textbf{Data Address Bus} - carries the address of data between main components.
    \item \textbf{Instruction Bus} - carries instructions between main components.
    \item \textbf{Instruction Address Bus} - carries address of instructions between main components.
\end{itemize}

Having individual buses instead of the same bus mitigates the \textit{Von Neumann bottleneck}, since execution of instructions can be performed in parallel.
\medskip

\textbf{Applications}

\begin{itemize}
    \item \textbf{Digital Signal Processors (DSPs)} - audio/video processing, telecommunications, radar systems, and image processing.
    \item \textbf{Microcontrollers (MCUs)} - embedded systems.
    \item \textbf{Network Processors} - routers, switches, and network security appliances.
    \item \textbf{Automotive Systems} - Engine control units, advanced driver-assistance systems, infotainment systems.
\end{itemize}

\subsection*{Flynn's Taxonomy}

\textbf{Flynn's Taxonomy} classifies computer architectures according to how many \textit{processes} they can process simultaneously.

\begin{itemize}
    \item \textbf{Single-Instruction, Single-Data (SISD)} - Uniprocessor machine capable of executable a single instruction operating on a single data stream.
        \begin{itemize}
            \item Instructions processed in a sequential manner.
            \item Computers called \textbf{sequential computers}.
            \item Speed of processing element is \textit{dependent} by the \textit{rate} at which the computer can transfer information internally.
        \end{itemize}
    \item \textbf{Single-Instruction, Multiple-Data (SIMD)} - Multiprocessor machine capable of executing the same instruction on all the CPUs but operating on different data streams.
        \begin{itemize}
            \item Well suited to \textit{scientific computing}.
            \item Involves lots of vector and matrix operations.
        \end{itemize}
    \item \textbf{Multiple-Instruction, Single-Data (MISD)} - Multiprocessor machine capable of executing different instructions on different PEs but all of them operating on the same dataset.
        \begin{itemize}
            \item Not useful in most applications.
        \end{itemize}
    \item \textbf{Multiple-Instruction, Multiple-Data (MIMD)} - Multiprocessor machine capable of executing multiple instructions on multiple data sets.
        \begin{itemize}
            \item Each PE has separate instruction and data streams.
            \item Versatile applicability.
            \item \textbf{Asynchronous}, unlike \textit{SIMD} and \textit{MISD}.
            \item Generally considered superior to other models \textit{(when applicable)}.
        \end{itemize}

        Can be classified into two models:
        \begin{itemize}
            \item \textbf{Shared-Memory (tightly coupled)} - All processors use the same global memory, through which communication occurs.\\
                They are \textit{easy} to program but \textit{hard} to scale and vulnerable to failure.
            \item \textbf{Distributed Memory (loosely coupled)} - Each processor has its own local memory. Communication occurs through an \textit{interconnection network}.\\
                More complicated to program but \textit{scalable} and \textit{fault-tolerant}.
        \end{itemize}
\end{itemize}

\section*{Chapter 2 - Data Representation}

\subsection*{Number Systems \textit{(Reiteration)}}

\textbf{Decimal System}

\begin{itemize}
    \item The standard system for denoting numbers in a \textit{human-readable} format.
    \item Also called the \textbf{base-10} system.
\end{itemize}

\textbf{Binary System}

\begin{itemize}
    \item Useful in electronic devices and compute systems because of easy performability.
    \item Also called the \textbf{base-2} system.
\end{itemize}

\textbf{Octal System}

\begin{itemize}
    \item Also called the \textbf{base-8} system.
    \item Used for representing file permissions in \textit{Unix/Linux} operating systems.
\end{itemize}

\textbf{Hexadecimal System}

\begin{itemize}
    \item Digits formed using 16 digits including \texttt{A}, \texttt{B}, \texttt{C}, \texttt{D}, \texttt{E}, and \texttt{F}.
    \item Also called the \textbf{base-16} system.
    \item Used for handling memory address.
\end{itemize}

\subsection*{Character Encoding Systems}

\textbf{Character encoding} is the process of converting characters into a format that computers can parse and store.
\medskip

\textbf{American Standard Code for Information Interchange (ASCII)}
    \begin{itemize}
        \item \textit{7-bit encoding}, limited to English characters and a basic set of symbols.
        \item \textit{Non-printable system codes} between 0 and 31.
        \item Lower ASCII between 32 and 127.
        \item Higher ASCII between 128 and 255.
    \end{itemize}

\textbf{Unicode}
    \begin{itemize}
        \item Universal encoding technique; can be used for any language or letter for any device.
        \item Used without generating data corruption in a variety of systems.
        \item Defines \textbf{code points}; they \textit{don't} define how code points are stored as sequence of bytes; that's where \textit{UTF} comes into play.
        \item \textbf{Unicode Transformation Formats (UTFs)} - Standardized methods for encoding Unicode characters into a sequence of bytes.
            \begin{itemize}
                \item \textbf{UTF-16} - Uses \textit{16-bit} code units.
                    \begin{itemize}
                        \item Character '\texttt{B}' (\texttt{U+0042}) - stored as \texttt{00 42}.
                        \item Used internally by operating systems and programming environments for string storage and text manipulation.
                    \end{itemize}
                \item \textbf{UTF-8} - Uses 1 to 4 \textit{8-bit} code units.
                    \begin{itemize}
                        \item Can encode all valid Unicode character code points.
                        \item Code points for \textit{lower} values are encoded with \textit{fewer} bytes.
                        \item \textbf{Backwards compatible} with \textit{ASCII}; Unicode characters encoded in \textit{UTF-8} that are \textit{one-to-one} with \textit{ASCII} are encoded using a single byte with an identical binary value.
                    \end{itemize}
            \end{itemize}
    \end{itemize}
